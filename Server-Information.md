## 3) Direct Capabilities
- The server can run smoothly with up to ***`6-8`*** players, meaning we will cap the players allowed in the server to 8.
- The server can run some intesnive but optimized machines, and up to ***`120`*** mods
- There will be some rules in place to be able to keep the full functionality and stability of the server in full working condition, but should not be too constricting.
- A maximum ***`2`*** players should be generating new chunks (flying into new territory) at the same time.
- No more than ***`200-300`*** entities should be rendered within a single location
---
## 4) Guidlines to remember
### This are not DIRECT rules (yet) but soemthing to keep in mind to keep the server up and smooth that MAY be turned into (or the baseline of) rules later on
- Turn Machines Off: If you aren't using a machine/auto-farm, turn it off when not in use.
- Don't leave items sitting on the ground. 
- Spread Out Bases: Don't build all 6 player bases in the same 4 chunks. While it’s fun to be neighbors, cramming all those mods and entities into one spot creates a "lag hole."
- No "Elytra Racing" in New Terrain: Ask players to stick to "Pre-Generated" areas (areas you've already visited) if they want to fly at high speeds.
- Do not go too far: While being farther apart can reduce server lag, spreadnig too far apart can have the game render different POV's and chunks for each active person, reducing preformance
